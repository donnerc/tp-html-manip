Avant de pouvoir afficher un document HTML, un navigateur Web doit pouvoir comprendre ce code HTML. Il va effectuer une analyse du code et transformer ce code linéaire en une structure d'arbre beaucoup plus pratique à gérer puisque le code HTML n'est rien d'autre qu'un emboîtement de balises les unes dans les autres.

Section 3 : génération de code HTML à partir d'un arbre

Une fois que l'on dispose de l'arbre qui représente le document HTML, il est non seulement très facile de le manipuler, mais également de générer le code HTML correspondant. Ceci est donc très utile pour modifier un fichier HTML existant.

Exemple de manipulations qu'il est possible d'effectuer

  • Extraire du fichier HTML tous les liens contenus dans le document. Il suffit pour ce faire de parcourir l'arbre de représentation et de s'intéresser aux noeuds qui référencent une balise <a>
  • Compter le nombre de mots qu'il y a dans le fichier HTML
  • Rechercher tous les titres du document et générer une table des matières
  • Générer une version texte (ascii) sans mise en forme
  • Corriger l'indentation du code HTML (L'indentation dépend alors du niveau dans l'arbre
